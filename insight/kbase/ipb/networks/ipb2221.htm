Problems Installing Windows NT 3.51 on XEN-PC (Jade) Systems

Problems have been found with installing Windows NT 3.51 on XEN-PC Jade systems.

The failure to install occurs in one of two ways depending upon the type of installation being attempted.

If installing Windows NT v3.51 onto a new system, once the three Installation Diskettes have been processed the installation fails and gives the message "Unable to Locate Hard Drive".

If attempting to perform an upgrade, once the installation process has copied all of the files required to perform the upgrade the installation fails and gives the message "Inaccessible Boot Device".

Microsoft have produced a fix for the problem. The fix can be downloaded from file Area 88 on Apricot Insight BBS. Filename - ATAPI.ZIP

To implement the fix:-

  1. Unpack the file ATAPI.ZIP to produce the file ATAPI.SYS - 12,336 bytes.

  2. Make a backup copy of the Windows NT setup disk 3 and copy the ATAPI.SYS file to the backup copy.

  3. Install Windows NT 3.51 using the backup copy of the setup disk 3.

    At the screen Mass Storage Devices, do not panic if no hard disks can be found. Windows NT is only looking for a CD-ROM controller or a SCSI controller. Press ENTER to continue with the installation.

  4. After the first part of the installation, Windows NT will re-boot.

    If Windows NT errors on boot up with a message Inaccessible boot device, the following steps also need to be applied:-

  5. Boot from an MS-DOS diskette.

  6. Copy the ATAPI.SYS file to the \WINNT35\SYSTEM32\DRIVERS directory.

  7. Re-boot the system, and the Windows NT 3.51 installation can now be completed.

The Windows NT 3.51 service pack 2 and above includes the updated version of the ATAPI.SYS file.

Apricot recommends that the latest service pack is applied to all installations of Windows NT 3.51. The latest service pack is available from Apricot Insight BBS, file area 88.
