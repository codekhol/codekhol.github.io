Novell NetWare V2.15 and EZTAPE

We have had a number of reports from resellers about people trying to use EZTAPE on Apricot Qi machines acting as NetWare 2.15 ELS servers (in non-dedicated mode), where EZTAPE would hang the server when run.

We have investigated this problem, and it only occurs if the NetWare shell used (NET2.COM, NET3.COM or NET4.COM) is NetWare V2.15 rev C. You can find out what version of NETx.COM you have by typing "Netx i " (where x is the major DOS version - 2, 3 or 4 ). The problem does not occur if you use the NetWare V2.15 rev A shell.

We have discussed this problem with Novell UK, and it appears that they have been aware of the problem for about two months now (it has been published in the Novell UK Support Technical Bulletins) and that it is caused by a clash between the rev C shell and Irwin's tape backup software (regardless of hardware platform). Novell UK recommend that the rev A shell be used to overcome this problem, and they also state that the use of the rev A will cause no other support issues.

Novell UK also stated that the problem is likely to occur on workstations using EZTAPE and the NetWare 2.15 rev C shell and that the fix is the same (i.e. use the rev A shell).

We have also tested the same configurations with the NetWare 386 V3.00 rev A shell and the problem does not occur, so it would appear that Novell have fixed the problem in later releases of shell.

The NetWare V2.15 rev A shell files (NET2.COM, NET3.COM and NET4.COM) are all available on the Apricot Technical Support Bulletin Board System in File Area 28.
