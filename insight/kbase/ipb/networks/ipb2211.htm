Mounting the Insight CD Release 7 as a NetWare Volume

It has been brought to our attention that the new Release 7 Insight CD has been created with a volume label of '_' (i.e. an underscore character). When mounting the CD as a NetWare volume under NetWare 3.12 or 4.x, the CDROM.NLM driver will reject this as an invalid name and will refuse to mount the volume.

There is however a work-around for this problem, which is as follows. Firstly, if running NetWare 3.12 or NetWare 4.0x, ensure that you have CDROM.NLM version 4.10s (this can be obtained by downloading CDROM3.EXE from the Insight CD Release 6, from the Insight BBS, or from Novell's NetWire forum on CompuServe) - IPB 2188 details relevant load instructions for this version. If using NetWare 4.10, the CDROM.NLM driver does NOT need upgrading.

With the appropriate CDROM.NLM driver loaded, find out what logical device number the CD-ROM drive is using by typing CD DEVICE LIST, and then use the CD RENAME command as follows (assuming that the CD logical number is 8, and the desired volume name for the CD is 'INSIGHT');-

CD RENAME /D=8 INSIGHT

You should then be able to mount the CD with the CD MOUNT command (by either specifying the new volume name or the logical device number).
