Novell NetWare Updates

A number of new NetWare updates have just been released by Novell. All the updates documented below can be found in File Area 37 of Apricot Insight BBS.

  • CDROM3.EXE
    This update contains a new version of the CD-ROM support modules for use with NetWare 3.12 and 4.01. The update replaces the files in CDROM2.EXE (see IPB 2183 ) and includes better support for CD-ROMs that have a large number of directory entries. Note however that the restriction documented in IPB 2188 still applies with this new version of CDROM.NLM (version 4.10s).

  • DS310.EXE
    This update contains an update that modifies the Directory Services module (DS.NLM) held within SERVER.EXE for NetWare 4.01. The update needs to be run at the DOS prompt on the NetWare 4.01 server, and will patch SERVER.EXE so that DS.NLM should now report 
    	DS.NLM
		NetWare Directory Services (310)
		Version 4.02		June 6, 1994.

    The new Directory Services update should improve performance and reliability when servers are synchronising with remote hosts, and also includes a number of bug-fixes. If installing this update, you should ensure that the version of DSREPAIR.NLM from the DSREPS.EXE update is also installed on the server.

  • NWADMN.EXE
    This update contains version 4.06 of the Windows NWADMIN program for NetWare 4.01. The new version includes a number of bug-fixes and feature enhancements, making NWADMIN easier to use, including the ability to print the NDS tree, and to save the current location in the NDS tree. If installing this version, DO NOT copy it over the existing NWADMIN files in \PUBLIC on the NetWare 4.01 server, or else NWADMIN will report an unknown error when loading. The new version should be installed to a new directory, and should be run from there.

All the above updates contain full documentation detailing the changes and fixes that have been made, along with full installation instructions. This documentation should be read and understood prior to installation. Apricot recommend that these updates be installed on ALL relevant servers.
