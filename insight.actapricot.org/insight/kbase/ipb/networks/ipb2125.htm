Windows for Workgroups 3.1 Issues

Microsoft have recently begun shipping a new version of Windows 3.1 that also contains peer-to-peer networking capabilities. Windows for Workgroups enables Windows users to communicate with one another across a LAN, whilst at the same time allowing connectivity to Novell NetWare and Microsoft LAN Manager networks.

Windows for Workgroups and LAN Card Driver Support

Windows for Workgroups as shipped by Microsoft includes support for about 100 different Microsoft certified LAN adapters, including the 3Com 3C503 and 3C523, and the Western Digital (and SMC) LAN cards. Apricot's 82592 based on-board Ethernet (as found on LS, XEN-LS, XEN-S, and Qi) is not included in the initial release of Windows for Workgroups, as the certification of the 82592 NDIS driver was completed too late for inclusion into the initial product. Microsoft will however be including the drivers in future maintenance updates to the product.

Users with Apricot systems that have on-board Ethernet therefore need to have a separate drivers disk for the on-board Ethernet drivers. All new LS Pro and XEN-LS II systems will be supplied with a suitable disk as standard, but users of the earlier 82592 based systems will not have a suitable drivers disk with their systems. Drivers for both types of on-board Ethernet can be found on the Apricot Bulletin Board System, in File Area 5. For the 82592 based systems, download ETH592.ZIP, and for the LS Pro and XEN-LS II, download ETH596.ZIP. Both files contain the NDIS driver needed for the appropriate systems, along with an OEMSETUP.INF file that Windows for Workgroups SETUP will use when loading the drivers.

ETH592.DOS and APRICOT.386

Users of 82592 based systems (LS, XEN-LS, XEN-S and Qi) will also need to make a change to their SYSTEM.INI files to enable the correct operation of the on-board Ethernet. In the [386Enh] section of SYSTEM.INI, add the following line:-

DEVICE=APRICOT.386

and copy APRICOT.386 from the ETH592.ZIP archive into the \WINDOWS\SYSTEM directory.

DEVICEHIGH=ETH596.DOS hangs NET START command

Users of LS Pro and XEN-LS II should note that the ETH596.DOS driver should NOT currently be loaded high (into Upper Memory), as this will cause the NET START command to hang the machine. This problem will be resolved in a future release of ETH596.DOS. Other drivers can however be loaded high without causing additional problems.

Apricot and support for Windows for Workgroups

Apricot will only be supplying Windows for Workgroups on LS Pro and XEN-LS II systems that have both on-board Ethernet, and a local hard-disk, and as such, will only fully support Windows for Workgroups as pre-installed on these systems. We will answer support calls on Windows for Workgroups on other Apricot hardware, but ONLY if the call is to do with Apricot specific issues (for example trouble installing the Apricot on-board Ethernet LAN drivers). Calls for general support on Windows for Workgroups as supplied by Microsoft (as opposed to the pre-installed version on the LS Pro and XEN-LS II) should be directed to Microsoft on 0734-271000. Apricot Technical Support will be checking the validity of all Windows for Workgroups calls, and will direct callers on to Microsoft if applicable.
