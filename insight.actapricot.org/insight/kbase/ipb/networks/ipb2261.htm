LS Pro (Caracal) losing network connection running Windows 95

Under certain circumstances in Windows 95, the Network Browser will not be able to locate any other nodes on the network.

This problem exhibits itself because Windows 95 does not currently correctly detect the AMD PCnet settings on some LS Pro systems.

To overcome this problem follow the steps below:

Instructions

  • Go into Control Panel in Windows 95 & double click on Network icon
  • You should now see the Network Configuration box
  • Highlight AMD PCNET Family Ethernet VL-Bus adapter and then click on properties button to your lower right.
  • In this Box you should make sure the radio button is highlighted on "Enhanced mode 32 bit and 16 bit) NDis Drivers.
  • Then click on Advanced "settings"
  • You should then see a box marked Property: [Make sure the values are as follows]:-
    • Bus Timer (micro seconds): Default
    • Bus Type: Vesa
    • Full Duplex: Default
    • LED0: c0
    • LED1: b0
    • LED2: 4088
    • LED3: 81
    • TP: Off
  • Then Click OK button.
  • If you have the NetWare client loaded follow these steps:
  • When you return to the main Network box menu, highlight the " ipx/spx - compatible protocol and then click on the properties button, then click on advanced "settings" and highlight Frame type and set this from Auto to the preferred frame type on your network I.E 802.3 (See IPB 2253).
  • Click OK to exit the network setup box and Windows 95 will ask you if you want to restart the computer.
  • Click on yes.
